Step 4   The bar law (commit 3a3226f). bar_rescale_updates builds the
         field->expression dict for ONE atomic .update(): both maxima
         recomputed from effective stats and both currents rescaled
         current × new_max ÷ old_max (Case/Greatest/Round-over-Cast),
         rounded to nearest, floored at 1 while alive, a dying 0 staying
         0, full bars staying exactly full. All stat-field references
         are F() (DB truth at update time); only the gear sums and
         spend's delta arrive as Python constants — they ARE the
         mutation. Adopters: equip_item / unequip_item (rescale inside
         their @database_sync_to_async bodies after the item write, with
         a status refresh pushed so the pane tracks the new maxima) and
         cmd_spend.apply_spend (stat bump + point debit + rescale in one
         .update(); field-limited refresh afterwards — a bare
         refresh_from_db cleared the current_room FK cache the async
         caller still reads, caught by the new tests). Level-up
         untouched per the brief (full-refill recalculate_bars path,
         now fed effective inputs).

Step 5b  The wiring (commit 790259b). PROC_CHANCE_PER_POINT 0.05 /
         PROC_CHANCE_CAP 0.50 / PROC_FACTOR_STATS constants;
         roll_gear_bonus_damage rolls per equipped item per proc-factor
         entry on landed hits only (hit/critical — a graze rolls
         nothing), success adds randint(1, ceil(V)); flat
         electric_damage_bonus joins every landed hit; lifesteal heals
         the summed rolled values via an atomic Least(F+heal, F(max))
         update, then refreshes the in-memory character and pushes a
         status — no output line; the four inert stats
         (spell_damage_bonus, mana_regen, magic_resist,
         radiation_resist) have no consumers and do not join TAV.
         NPC->player damage never gains procs.

         CRIT-BONUS APPROACH: resolve_hit gained an optional
         crit_bonus=0.0 keyword parameter added inside the existing
         capped computation (min(CRIT_CAP, max(CRIT_BASE, base + dex
         advantage + crit_bonus))). The player-attack site passes
         summed gear crit_chance × 0.01; the NPC-attack site passes
         nothing — signature stays workable for every existing caller
         and the cap provably holds (tested with an absurd bonus).

         DISPLAY: the gear pool renders as one parenthetical on the hit
         line — "You hit X for 14 (+7) damage." / "for a critical 14
         (+7) damage!" — base first, total dealt = base + bonus, zero
         pool byte-identical to pre-B5 (verified by test). The
         parenthetical inherits the hit line's category color; no new
         palette entries.
