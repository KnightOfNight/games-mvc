What shipped
------------
Server (django/src/apps/shyland/consumers.py):
  - connect(): a 'superseded' broadcast to the character's personal
    player_{pk} group, fired immediately after the session_token mint and
    before the unconditional presence SET. Payload: type=player_message,
    event=superseded, token=<new session_token>, ts=envelope_ts(). No
    text/category/status fields.
  - player_message(): a 'superseded' branch checked first, before
    clear/text/status/fight delivery and the dying/respawn events.
    Token match -> return (own broadcast). Mismatch -> authored farewell
    ("The world's attention shifts — your story is being told through
    another window now. This one falls quiet.", category system), then
    {'event': 'superseded', 'ts': ...} (matching cmd_quit's quit-event
    shape — the shape in code carries no 'type' field, per the brief's
    match-the-code instruction), then a normal await self.close(). The
    existing disconnect() path handles the guarded presence delete, group
    discards, heartbeat cancellation, and last_seen — nothing duplicated.
  - Not a command: passes through no command gates (fires while dying and
    in combat). No room broadcast. No models, no migration, no tick-engine,
    presence-Lua, or disconnect() changes.

Client (django/src/apps/shyland/templates/shyland/game.html):
  - 'superseded' flag parallel to 'quitting'; set on the superseded event.
  - Close handler: when set, suppresses the reconnect placeholder AND the
    auto-reconnect loop (which would otherwise retake the session), and
    does nothing else — no navigation, no reload. Farewell stays visible;
    refresh-to-retake works. A close without superseded/quit is unchanged.
