CONFORMANCE DELTAS AND JUDGMENT CALLS (beyond Step 4's list)
------------------------------------------------------------
1.  attack's pool remains living NPCs only — DD §8 names players, but
    no PvP mechanics exist to attack a player with; implementing the
    refusal would invent design. Examine's union DOES include players.
    Flag for the design chat if a PvP placeholder refusal is wanted.
2.  buy: the funds check stays atomic all-or-nothing (DD §3 itself
    files "can't afford" under mechanical failure/warn); the STOCK
    shortfall is partial by analogy (buys what's left + "They only had
    N."). Bare-numeric 'buy 3' answers "buy 3 what?" per fn 13.
3.  Loot's per-item "You loot …" lines stay success/value-colored —
    the DD §2 loot-color enumeration does not name them (only pickup
    lines); flag if they should join loot-color.
4.  Deleting #8A887F "from the file entirely" (Step 2.4) re-pointed
    .msg-system, .msg-room, .msg-echo, and --loc-sep to var(--muted)
    (#6b6b80) — a slight visible shift on system/ambient lines and the
    location-bar separators.
5.  The tick engine's dying fatal-blow line keeps category 'error' —
    the v19 brief-4 ruling ("red fatal-blow line") predates and
    survives the three-layer doctrine; error-color is the same red.
6.  pickup-all's oldest-first order uses created_at/pk — the best
    existing signal; there is no floor-arrival timestamp (a
    player-dropped old item sorts by its mint time). No field added.
7.  list's Details for vendor stock renders "100%, Common" (durability
    items) / "Common" (others); no bind word — entries are definitions,
    not instances. Quantity shows remaining stock for limited entries,
    blank for unlimited.
8.  spend/loot/repair bare invocations now prompt (fn 10), replacing
    the old conveniences (spend->stats, loot->first corpse,
    repair->first damaged). bare timestamps now reports the current
    setting (DD §10) instead of erroring.
9.  equip's sentence no longer names the slot (DD §6); the paper-doll
    carries slot placement.
10. Two v20 grammar-table cases and one dispatch-guard case were
    updated where the chart supersedes them (sell partial fulfillment,
    drop numeric-only, unknown-command error layer), plus
    test_room_visits' revisit case now sets brief_mode explicitly
    (default flipped). The v20 "add cases, change none" law yields to
    the DD by the brief's own authority.
11. who's empty-world line ("No players are currently online.") is
    retained for the unreachable zero case (the asker is always
    present in practice).
12. Say completes no argument options (free text); the fn-9/10 pairing
    means bare say prompts and anything else passes through.
